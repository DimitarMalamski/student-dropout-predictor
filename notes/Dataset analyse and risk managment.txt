Other risks and potential solutions for them:
1. Data Leakage:
	- the dataset contains a lot of information, but the main concern is the fact that it contains final outcome metrices ( plus 2nd semester as well ), which is a problem for the training of the model.
	* In order to battle with this issue, I need to exclude those features and work with the rest of the data ( only metrices regarding semester 1 );

2. Class imbalance:
	- In the dataset, the ratio between “Graduates” and “Dropouts” is not distributes symmetrically, which may result in the accuracy becoming misleading.
	* In order to battle this I need to check Class Distribution early or even tune up the data beforehand for proper use;

3. Ethical & Fairness risks:
	- The dataset contains values like: gender, nationality, parents’ occupation - which may result in different types of bias in the model ( socioeconomic bias, gender bias, cultural bias... );
	* Discussion regarding the inclusion of those features should be made beforehand to evaluate whether I should use them in this Challenge or not;

4. Overcomplexity:
	- The dataset is rich in different metrices, which may result in overcomplicating things from the start;
	* One tactic about this is to start simply ( Initially to use “Admission data” and later to include 1st Semester metrices );

5. Timeframe:
	- There is a possibility the timeframe for the Practice Challenge might not be enough, since a lot of cleaning and encoding of data has to be made.
	* This also needs to be communicated if it is realistic to do so

6. Generalizability Risk
	- The dataset is gotten from a Portuguese university, which may result is poor outcomes on other institutions
